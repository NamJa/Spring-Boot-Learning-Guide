# 부록 C. AOP와 프록시 — 보이지 않는 마법의 정체

로깅·트랜잭션·보안처럼 **여러 메서드에 공통으로 끼어드는 관심사를 비즈니스 코드에서 분리하는 기술**이 바로 AOP(Aspect-Oriented Programming, 관점 지향 프로그래밍)입니다.

이미 여러분은 AOP를 쓰고 있었습니다. Phase 3-4에서 만난 `@Transactional`은 사실 **Spring AOP가 만든 프록시**가 메서드 앞뒤에서 트랜잭션을 열고 닫아 주는 것이고, Spring Security의 `@PreAuthorize` 같은 권한 검사도 동일한 메커니즘으로 동작합니다. "애너테이션 하나 붙였을 뿐인데 마법처럼 동작하는" 그 정체가 AOP입니다.

이 부록은 김영한 님의 **"스프링 핵심 원리 - 고급편"** 의 프록시·AOP 파트를 Kotlin과 도서 API 예제로 압축해 다룹니다. 본문에서 `@Transactional`로만 살짝 스쳤던 내용을 한 단계 깊이 파고듭니다.

## 1. 횡단 관심사(cross-cutting concern)란?

애플리케이션의 관심사는 두 부류로 나뉩니다.

- **핵심 관심사(core concern)**: "도서를 저장한다", "주문 금액을 계산한다" 같은 **비즈니스 로직** 그 자체.
- **횡단 관심사(cross-cutting concern)**: 로깅, 트랜잭션, 보안, 성능 측정, 캐싱처럼 **여러 비즈니스 로직을 가로질러 공통으로 필요한** 부가 기능.

이름이 "횡단"인 이유는, 아래처럼 수직으로 쌓인 비즈니스 메서드들을 **가로로 관통**하기 때문입니다.

```
              create()   findById()   update()   delete()
              ────────   ──────────   ────────   ────────
  로깅      ───┼───────────┼────────────┼──────────┼───►  (횡단)
  트랜잭션  ───┼───────────┼────────────┼──────────┼───►  (횡단)
  보안      ───┼───────────┼────────────┼──────────┼───►  (횡단)
              비즈니스      비즈니스      비즈니스    비즈니스
```

이 횡단 관심사를 비즈니스 메서드마다 손으로 복사해 넣으면, 코드가 중복되고 핵심 로직이 부가 코드에 파묻힙니다. AOP는 이를 **한 곳(Aspect)에 모아 두고, 정해진 지점에 자동으로 끼워 넣는** 방식으로 해결합니다.

## 2. 이미 쓰고 있던 AOP

| 기능 | 사용한 애너테이션 | 실제 동작 |
| --- | --- | --- |
| 트랜잭션 | `@Transactional` | 프록시가 메서드 전후로 트랜잭션 시작/커밋/롤백 |
| 보안 | `@PreAuthorize`, `@Secured` | 프록시가 메서드 호출 전 권한 검사 |
| 캐싱 | `@Cacheable` | 프록시가 캐시 조회 후 있으면 메서드 자체를 건너뜀 |
| 비동기 | `@Async` | 프록시가 별도 스레드에서 메서드 실행 |

이 모든 기능의 공통점은 **"여러분이 짠 객체를 Spring이 한 겹 감싼다(프록시한다)"** 는 것입니다. 부록 C는 그 한 겹의 실체를 다룹니다.

## 4. 선수 지식

- **Phase 0** IoC/DI와 빈(bean) 개념 — AOP는 "빈을 프록시로 바꿔치기"하는 기술입니다.
- **Phase 1** `kotlin("plugin.spring")`(all-open) 플러그인 — Kotlin의 `final` 기본값과 프록시의 충돌을 이미 해결해 둔 설정입니다.
- **Phase 3-4** [`@Transactional`](../phase-3-data-jpa/04-transactions.md) — 우리가 처음 만난 AOP 적용 사례이자, 자기 호출 함정의 출발점입니다.

> [!TIP]
> AOP는 "원리를 모른 채 써도 동작하지만, 원리를 모르면 디버깅을 못 하는" 대표적인 영역입니다. 자기 호출로 `@Transactional`이 안 먹는 버그를 한 번이라도 겪어 본 사람은 이 부록의 가치를 압니다.
